# N8N WORKFLOW TESTING GUIDE
# ===========================

## COMPREHENSIVE TESTING PROCEDURE
## ================================

## ================================================================
## PHASE 1: PRE-TESTING SETUP
## ================================================================

### STEP 1: ENVIRONMENT PREPARATION
□ All environment variables set correctly
□ n8n running and accessible
□ Both workflows imported successfully
□ Google Sheets created and shared
□ AI model deployed and responding
□ Angel One API credentials valid

### STEP 2: PAPER TRADING MODE
CRITICAL: Always start with paper trading!

Set these environment variables:
N8N_PAPER_TRADING_MODE=true
N8N_LOT_SIZE=1  # Start with minimum
N8N_MAX_DAILY_LOSS=100  # Low limit for testing

### STEP 3: BACKUP ORIGINAL WORKFLOWS
1. Export both workflows
2. Save as backup files
3. Document any customizations

## ================================================================
## PHASE 2: INDIVIDUAL NODE TESTING
## ================================================================

### TEST 1: CRON TRIGGER
1. Open main workflow
2. Click "Trading Hours Cron" node
3. Click "Execute Node"
4. Should trigger during trading hours only
5. Check execution log

EXPECTED RESULT: Node executes during 9:15 AM - 3:15 PM IST

### TEST 2: TIME FILTER
1. Click "Trading Hours Filter" node
2. Execute with current time
3. Should pass during trading hours
4. Should block outside trading hours

EXPECTED RESULT: Conditional logic works correctly

### TEST 3: ANGEL ONE LOGIN
1. Click "Angel One Login" node
2. Execute node
3. Check response for JWT token
4. Verify no authentication errors

EXPECTED RESULT:
{
  "status": true,
  "message": "SUCCESS",
  "data": {
    "jwtToken": "eyJ...",
    "refreshToken": "...",
    "feedToken": "..."
  }
}

TROUBLESHOOTING:
- Invalid TOTP: Update with current code
- API Key error: Check credentials
- Network error: Check internet connection

### TEST 4: MARKET DATA NODES
Test each market data node:

SPOT PRICE NODE:
1. Click "Get Spot Price"
2. Execute node
3. Should return current BANKNIFTY price

EXPECTED RESULT:
{
  "status": true,
  "data": {
    "exchange": "NSE",
    "tradingsymbol": "BANKNIFTY-INDEX",
    "ltp": 45250.75
  }
}

CANDLE DATA NODE:
1. Click "Get 5Min Candles"
2. Execute node
3. Should return array of candle data

VIX DATA NODE:
1. Click "Get India VIX"
2. Execute node
3. Should return VIX value

### TEST 5: TECHNICAL INDICATORS
1. Click "Calculate Technical Indicators"
2. Ensure candle data is available
3. Execute node
4. Check calculated values

EXPECTED RESULT:
{
  "indicators": {
    "rsi": 45.67,
    "macd": {
      "macd": 1.23,
      "signal": 0.89,
      "histogram": 0.34
    },
    "momentum": 2.45,
    "volumeRatio": 1.34,
    "candlePattern": "NORMAL"
  }
}

### TEST 6: AI MODEL
1. Click "AI Trade Confirmation"
2. Execute with sample data
3. Check response format

EXPECTED RESULT:
{
  "signal": "BUY_CE",
  "confidence": 0.84,
  "timestamp": "2024-01-15T10:30:00"
}

TROUBLESHOOTING:
- Model not responding: Check deployment
- Invalid response: Verify input format
- Timeout: Check model performance

### TEST 7: GOOGLE SHEETS
1. Click any Google Sheets node
2. Execute with test data
3. Check sheet for new row

EXPECTED RESULT: New row appears in Google Sheet

## ================================================================
## PHASE 3: WORKFLOW INTEGRATION TESTING
## ================================================================

### TEST 1: FULL WORKFLOW EXECUTION
1. Set paper trading mode
2. Execute entire workflow manually
3. Monitor each node execution
4. Check for errors or failures

### TEST 2: DATA FLOW VALIDATION
Verify data passes correctly between nodes:
1. Login → Market Data (JWT token)
2. Market Data → Indicators (price data)
3. Indicators → AI Model (calculated values)
4. AI Model → Order Logic (signal/confidence)

### TEST 3: CONDITIONAL LOGIC
Test all conditional nodes:
- Time filter (trading hours)
- VIX filter (< 18)
- Signal filter (not HOLD, confidence ≥ 0.75)

### TEST 4: ERROR HANDLING
Simulate errors and test recovery:
- Invalid TOTP code
- Network timeouts
- API rate limits
- Invalid market data

## ================================================================
## PHASE 4: PAPER TRADING VALIDATION
## ================================================================

### TEST 1: PAPER ORDER SIMULATION
1. Enable paper trading mode
2. Let workflow run for full cycle
3. Check Google Sheets for logged trades
4. Verify no real orders placed

### TEST 2: RISK MANAGEMENT
Test risk controls:
- VIX threshold (should skip when VIX > 18)
- Confidence threshold (should skip when < 0.75)
- Daily loss limits
- Position limits

### TEST 3: LOGGING VERIFICATION
Check all sheets are updated:
- Signals sheet: AI predictions logged
- Trades sheet: Paper trades recorded
- Active_Exit_Orders: SL/Target orders tracked

### TEST 4: PERFORMANCE MONITORING
Monitor for 1 week:
- Execution frequency (every 5 minutes)
- Success rate (% of successful executions)
- Error frequency
- Data accuracy

## ================================================================
## TROUBLESHOOTING COMMON ISSUES
## ================================================================

### ISSUE: Workflow not executing
SOLUTIONS:
1. Check cron expression
2. Verify workflow is active
3. Check time zone settings
4. Review execution logs

### ISSUE: Authentication failures
SOLUTIONS:
1. Update TOTP code
2. Check API credentials
3. Verify JWT token handling
4. Test manual login

### ISSUE: Data not logging to sheets
SOLUTIONS:
1. Check Google Sheets credentials
2. Verify sheet permissions
3. Test sheet connection
4. Check data format

### ISSUE: AI model not responding
SOLUTIONS:
1. Check model deployment
2. Verify API endpoint
3. Test with curl
4. Check model logs

### ISSUE: Orders not placing (paper mode)
SOLUTIONS:
1. Verify paper trading mode
2. Check order simulation logic
3. Review trade conditions
4. Test order parameters
