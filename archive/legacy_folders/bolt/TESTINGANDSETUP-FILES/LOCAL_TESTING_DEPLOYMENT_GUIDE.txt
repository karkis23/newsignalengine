## 1. LOCAL SYSTEM SETUP
## ================================================================

### STEP 1: INSTALL PYTHON
**For Windows:**
1. Go to https://python.org
2. Download Python 3.9 or higher
3. During installation, CHECK "Add Python to PATH"
4. Click "Install Now"
5. Open Command Prompt and verify: `python --version`

**For Mac:**
1. Install Homebrew: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
2. Install Python: `brew install python`
3. Verify: `python3 --version`

**For Linux:**
```bash
sudo apt update
sudo apt install python3 python3-pip
python3 --version
```

### STEP 2: CREATE PROJECT DIRECTORIES
```bash
# Create main project folder
mkdir trading-bot-apis
cd trading-bot-apis

# Create separate folders for each API
mkdir ai-model-api
mkdir sentiment-api
```

### STEP 3: SETUP AI MODEL API
```bash
cd ai-model-api

# Copy files (provided in this guide)
# - ai_model_api_fixed.py
# - requirements_fixed.txt

# Install dependencies
pip install -r requirements_fixed.txt

# For Mac/Linux users
pip3 install -r requirements_fixed.txt
```

### STEP 4: SETUP SENTIMENT API
```bash
cd ../sentiment-api

# Copy files (provided in this guide)
# - market_sentiment_api.py
# - sentiment_requirements.txt

# Install dependencies
pip install -r sentiment_requirements.txt

# For Mac/Linux users
pip3 install -r sentiment_requirements.txt
```

## ================================================================
## 6. TROUBLESHOOTING
## ================================================================

### COMMON ISSUES:

**Issue: "Module not found" error**
Solution:
```bash
pip install --upgrade pip
pip install -r requirements_fixed.txt
```

**Issue: "Port already in use"**
Solution:
```bash
# Kill process on port 5000
lsof -ti:5000 | xargs kill -9

# Or use different port
python ai_model_api_fixed.py --port 5002
```

**Issue: "News API quota exceeded"**
Solution:
- Check your NewsAPI usage at https://newsapi.org
- Wait for quota reset (daily)
- Consider upgrading NewsAPI plan

**Issue: "Connection refused" in n8n**
Solution:
- Ensure APIs are running
- Check ngrok URLs are correct
- Verify firewall settings

### DEBUGGING TIPS:
1. Check console logs for error messages
2. Test endpoints individually
3. Verify JSON format in requests
4. Check API response status codes
5. Monitor resource usage
